TinyClaw Core Model Refactor v1

本报告沉淀 TinyClaw 新一轮 core model 共识：第三方平台逻辑外挂， clawman 只接收标准化消息、调度 invocation、生成 delivery outbox。 本文不覆盖 clawman 内部 sandbox 模块重设计。

状态：目标模型草案 范围：Core Model Sandbox：单独设计 Token：CLAWMAN_API_TOKEN
1. 系统边界 2. 核心概念 3. 表模型 4. Inbound API 5. Dispatch 规则 6. Delivery Pull API 7. 关键决策 8. 暂不覆盖

1. 系统边界

TinyClaw core 不再直接持有企业微信、微信、斗鱼等第三方平台协议逻辑。 第三方平台协议、凭据、cursor、offset、重连状态都属于外挂 Channel Adapter。

Third-party Platforms - WeCom - WeChat - Douyu | v Channel Adapters - normalize inbound messages - own platform credentials/cursors - poll deliveries and send externally | v clawman core - rooms - messages - invocations - deliveries | v Sandbox module - designed separately

Channel Adapter 负责

  • 拉取或接收平台消息。
  • 持有平台凭据、cursor、offset、重连状态。
  • 标准化 inbound message 后调用 clawman。
  • 轮询 clawman deliveries，并执行真实外发。

clawman 负责

  • 接收标准化 inbound message。
  • 自动创建 Room，保存 Message。
  • 应用 Trigger Policy。
  • 创建和推进 Invocation。
  • 生成 Delivery outbox。

2. 核心概念

概念 定义 第一版约束
Room TinyClaw 内部会话容器。 一个外部 Channel Room 严格映射到一个 Room。
Message TinyClaw 内部入站事实，属于一个 Room。 不使用企业微信 archive seq 作为身份。
Invocation 一次 agent 执行。 第一版直接挂 Room；同一 Room 同时最多一个 active Invocation。
Delivery Invocation 产生的外发项。 发送成功与 Invocation 完成解耦。
Trigger Policy 决定 inbound Message 是否触发 Invocation 的规则。 Room 级可覆盖；为空时使用 channel default。

4. Inbound API

Adapter 调用 clawman 写入标准化消息。第一版统一使用 CLAWMAN_API_TOKEN。

POST /api/inbound
Authorization: Bearer <CLAWMAN_API_TOKEN>
{
  "channel": "wecom",
  "channel_room_id": "room-123",
  "channel_room_type": "group",
  "source_message_id": "external-msg-123",
  "sender_id": "user-1",
  "sender_name": "Alice",
  "message_time": "2026-05-19T10:00:00Z",
  "payload": {
    "type": "text",
    "text": "hello"
  }
}
  1. 使用 tenant_id = default。
  2. 按 (tenant_id, channel, channel_room_id) upsert Room。
  3. 按 (room_id, source_message_id) 幂等插入 Message。
  4. 如果 Message 已存在，返回已有结果，不重复触发。
  5. 应用 Room Trigger Policy；为空时使用 channel default rule。
  6. 根据 dispatch 规则创建或追加 Invocation。

5. Dispatch 规则

基础状态

dispatch_state 语义 说明
0waiting/context candidate未触发但可作为上下文。
1skipped永远不应进入 agent invocation。
>=1000attached绑定到真实 Invocation。
未触发不等于 skipped。群聊里未触发的普通消息保持 dispatch_state = 0， 后续触发时可作为上下文进入 Invocation。

无 active Invocation

  1. 新 Message 初始写入 dispatch_state = 0。
  2. 如果消息触发执行，创建 status = queued 的 Invocation。
  3. 把同一 Room 中 dispatch_state = 0 的 Messages 绑定到该 Invocation。
UPDATE messages
SET dispatch_state = $invocation_id
WHERE room_id = $room_id
  AND dispatch_state = 0;

已有 active Invocation

Active Invocation 只包括 queued 和 running。 当 Room 已有 active Invocation，新 Message 如果不是 skipped，直接绑定到该 Invocation。 

messages.dispatch_state = active_invocation.id

第一版只做追加，不做中断。

Invocation 状态

queued -> running -> completed
queued -> failed
running -> failed
queued/running -> cancelled

Invocation 创建后由 clawman 进程内 scheduler 启动执行，不设计外部 claim API。 scheduler 将 queued 推进到 running，调用当前配置的 agent runner； runner 成功时完成 Invocation，失败时标记 failed 并生成失败 Delivery。

failed 是终态；第一版不自动重试 failed Invocation。Invocation 失败时，clawman 创建一条失败提示 Delivery，后续用户重新触发会创建新的 Invocation。

6. Delivery Pull API

Adapter 轮询 clawman deliveries，并自行负责真实外发和短期重试。 Adapter 使用 deliveries.id 作为轮询游标。 

GET /api/deliveries?channel=wecom&id=123
Authorization: Bearer <CLAWMAN_API_TOKEN>

成功发送后 ack：

POST /api/deliveries/{id}/ack
Authorization: Bearer <CLAWMAN_API_TOKEN>
  • ack 后 Delivery 保留，只更新状态。
  • 第一版由 adapter 负责发送失败后的短期重试。
  • clawman 不主动重试 failed Delivery。

7. 关键决策

决策 结论
第三方平台逻辑 从 clawman 中剥离，由外挂 Channel Adapter 持有。
Room 创建 Inbound message 自动创建 Room。
tenant 第一版单租户，固定 tenant_id = default。
幂等 Adapter 必须提供 source_message_id。
Trigger clawman 根据 Room Trigger Policy 判断；adapter 不负责。
Agent Session 第一版隐式存在，不建 agent_sessions 表。
Delivery Invocation 完成不等待 Delivery ack。

8. 暂不覆盖

  • clawman 内部 sandbox 模块。
  • sandbox lifecycle / runtime connection。
  • Tool Runtime Backend：后续优先按“clawman 持有 agent loop，sandbox 只执行工具调用”设计。
  • durable workflow engine：第一版先用 invocations.status、input_snapshot、output_snapshot 和 dispatch_state 承载恢复语义。
  • memory capability。
  • explicit send capability。
  • 同一 Room 多个独立 Agent Session。
  • channel adapter 自身 cursor / offset / reconnect state。
