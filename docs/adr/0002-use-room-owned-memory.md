# Use Room-Owned Memory

TinyClaw stores durable memory as **Room Memory** owned by a **Room**, not by an **Agent Session**, Agent definition, runner session, or sandbox. The Room's default **Agent Session** may search **Room Memory** during **Agent Runs** and may produce **Memory Write Proposals** after a run, but Clawman governs scope, audit, and persistence. Run-scoped **Subagents** may read through the current Agent Run's memory capability, but they do not own or directly write Room Memory. This keeps long-term knowledge aligned with the collaboration container while allowing runners such as Codex or Claude to remain replaceable.
